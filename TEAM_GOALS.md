# SmartParking 各模块接下来的目标

基于项目当前已完成的功能，规划每个模块下一阶段的开发目标。

---

##  A — 基础架构

### 现状

项目骨架已搭好：CMake 多文件编译、MySQL 连接池、配置管理、路由注册全跑通。当前能正常启动和响应请求。

### 接下来的目标

#### P0 — CI/CD 流水线

- [ ] 创建 `.github/workflows/build.yml`，每次 push 自动编译检查
- [ ] 支持 Windows + Linux 双平台构建
- [ ] 加入 clang-tidy 或 cppcheck 静态检查

#### P0 — 日志系统

现在日志全是 `std::cout`，没有分级和文件输出：

```cpp
// 现状
std::cout << "[OK] 服务启动成功\n";
std::cout << "[WARN] 数据库连接失败\n";

// 目标
Logger::info("服务启动成功");
Logger::warn("数据库连接失败: {}", error_msg);
Logger::error("查询异常: {}", sql);
```

要求：
- 日志级别：DEBUG / INFO / WARN / ERROR
- 输出到控制台 + 文件（`logs/` 目录）
- 每条日志带时间戳
- 生产环境自动屏蔽 DEBUG 日志

#### P1 — 配置系统增强

- [ ] 支持环境变量覆盖配置文件（方便 Docker 部署）
- [ ] 新增 `--port` / `--config` 命令行参数
- [ ] 区分开发/生产配置

#### P1 — Docker 化

- [ ] 编写 `Dockerfile`（多阶段构建）
- [ ] 编写 `docker-compose.yml`（含 MySQL + 应用）
- [ ] 写一份 Docker 部署文档

#### P2 — 单元测试框架

- [ ] 引入 Google Test 或 Catch2
- [ ] 给 `base_service.h` 的 SQL 辅助函数写测试
- [ ] 给 `calculateFee` 写测试（这个逻辑复杂最容易出 bug）

#### P2 — 代码规范工具

- [ ] 加入 `.clang-format`，统一代码风格
- [ ] 加入 `.clang-tidy`，自动检查常见问题
- [ ] 在 CI 中检查代码格式

##  B — 用户与权限

### 现状

登录/注册/登出、Token 鉴权（24h TTL）、用户 CRUD、四级角色权限已实现。个人中心可查看余额和月卡。

### 接下来的目标

#### P0 — 密码重置功能

当前没有找回密码功能，用户忘记密码只能找管理员手动改：

```bash
# 新增 API
POST /api/auth/forgot-password  # 发送重置验证
POST /api/auth/reset-password   # 重置密码
```

（初期可以简单实现：通过预留的手机号 + 验证码，或用管理员后台重置）

#### P0 — Token 刷新机制

当前 Token 24 小时过期后必须重新登录：

- [ ] 新增 refresh_token 机制，access_token 过期后自动续期
- [ ] 前端检测到 401 时自动刷新 Token，对用户透明

#### P1 — 登录安全增强

- [ ] 登录失败次数限制（连续 5 次失败锁定 15 分钟）
- [ ] 保存登录日志（IP、时间、设备）
- [ ] 支持查看最近登录记录

#### P1 — 用户注册增强

- [ ] 注册时手机号格式校验（正则）
- [ ] 密码强度检查（长度、大小写、特殊字符）
- [ ] 用户名唯一性实时检查（输入框失焦即检查）

#### P2 — 多因素认证（可选）

- [ ] 手机短信验证码登录
- [ ] 管理员操作二次确认

#### P2 — 用户分组

当前只有角色，可以再加分组概念，比如 "月卡用户组" 可以批量设置优惠。

##  C — 停车核心

### 现状

车辆入库/出库、费用计算、容量原子更新、停车记录查询/导出、月卡自动免费通行都已实现。事务和原子操作也已加上。

### 接下来的目标

#### P0 — 车辆在场时长实时显示

Dashboard 上显示每辆车当前已停放时长：

```javascript
// dashboard.js
// 目标：在「我的车辆」卡片或车辆状态中显示
京A12345  停放中  已停 2小时15分
京B67890  空闲
```

后端需要新增一个计算时长的字段或 API。

#### P0 — 在场车辆列表（操作员/管理员视角）

当前 `/api/vehicle/status` 返回所有车辆的状态，但前端没有一个专门的「在场车辆」页面：

- [ ] Dashboard 新增「在场车辆」卡片
- [ ] 显示车牌、入库时间、已停时长
- [ ] 点击可直接出库
- [ ] 支持搜索筛选

#### P1 — 多停车场支持

数据库已经预留了多停车场字段（PARKING_LOT 表），但前端和管理功能还未完整支持：

- [ ] 管理页面可以添加/删除停车场
- [ ] Dashboard 可以切换查看不同停车场
- [ ] 出入库时选择目标停车场
- [ ] 统计报表区分不同停车场

#### P1 — 停车记录分页

当前 `/api/vehicle/query` 只返回最近 500 条：

- [ ] 加入 `page` 和 `page_size` 参数
- [ ] 前端增加分页控件
- [ ] 返回总记录数，支持跳转页码

#### P2 — 车位可视化地图

目前只有饼图展示占用率：

- [ ] 在 Dashboard 用网格展示每个车位（绿色=空闲，红色=占用，黄色=预约）
- [ ] 点击车位格子可查看当前车辆信息
- [ ] 根据停车场容量自动生成网格布局

#### P2 — 特殊车辆通道

- [ ] 军车、警车、消防车入库时自动识别放行
- [ ] 记录特殊车辆出入日志

##  D — 计费与余额

### 现状

原子扣款/充值/退款、计费规则 CRUD（标准/阶梯/会员/特殊）、月卡管理、套餐购买、自助充值已实现。

### 接下来的目标

#### P0 — 自助充值订单记录

当前自助充值直接到账，没有订单概念：

- [ ] 充值生成订单号（`CH` + 时间戳 + 随机数）
- [ ] 记录充值订单状态（待支付/已支付/已退款）
- [ ] 用户在个人中心可查看充值历史

#### P0 — 阶梯计费前端配置优化

当前阶梯计费需要在文本框手写 JSON：

```json
[{"hours":2,"rate":5},{"hours":4,"rate":3}]
```

- [ ] 改成可视化配置：点击"添加阶梯"自动生成行
- [ ] 每行输入「小时数」和「费率」
- [ ] 自动生成 JSON 存入数据库
- [ ] 编辑时自动解析 JSON 回填表单

#### P1 — 计费预览功能

用户在入库时选择计费方式，如果能预览费用会更好：

- [ ] 新增 `POST /api/parking/estimate-fee` API
- [ ] 传入车牌、计费方式、预估时长，返回预估费用
- [ ] Dashboard 入库时可以点击「预估费用」

#### P1 — 优惠券/折扣码

- [ ] 新增优惠券表（COUPON）
- [ ] 管理员可创建折扣码（满减/折扣率/有效期）
- [ ] 充值时或支付时输入折扣码享受优惠

#### P2 — 月度账单

- [ ] 每月 1 日自动生成上月的消费账单
- [ ] 包含：总消费、停车次数、日平均费用
- [ ] 在个人中心可查看历史账单

#### P2 — 发票功能

- [ ] 用户可申请开票（选择消费记录、填写发票抬头）
- [ ] 管理员后台可审核/导出发票

##  E — 增值功能

### 现状

预约（创建/取消/预付）、黑名单、收入汇总/趋势图、公告编辑已实现。

### 接下来的目标

#### P0 — 预约提醒功能

当前预约过期自动删除，但用户不知道快过期了：

- [ ] 预约成功时，前端倒计时显示剩余有效时间（精确到秒）
- [ ] 剩余 5 分钟时页面提示（弹窗或颜色变化）
- [ ] 超时后自动刷新页面状态

#### P0 — 报表导出

当前报表只在页面上展示数字：

- [ ] 增加「导出报表」按钮
- [ ] 导出为 Excel（`.xlsx`）或 CSV
- [ ] 包含：日期、收入、停车次数、套餐销售数
- [ ] 支持自定义日期范围

#### P1 — 黑名单入库拦截详情

当前黑名单车辆入库时只提示"已被列入黑名单"：

- [ ] 入库拦截时记录拦截日志（时间、车牌、原因）
- [ ] 管理员可查看拦截记录
- [ ] Dashboard 显示近期拦截统计

#### P1 — 预约历史

当前只显示 active 状态的预约：

- [ ] 新增「历史预约」标签页
- [ ] 显示已过期、已完成、已取消的预约
- [ ] 可按日期范围筛选

#### P2 — 收入预测（简单版）

基于历史数据做简单预测：

- [ ] 统计过去 30 天的日平均收入
- [ ] 显示本月预计总收入 = 日均收入 × 剩余天数
- [ ] Dashboard 上展示预测卡片

#### P2 — 公告增强

- [ ] 支持多条公告（当前只有一条）
- [ ] 公告支持 markdown 格式
- [ ] 公告可以设置有效期（过期自动隐藏）
- [ ] 管理员可置顶重要公告

##  F — 前端基础设施

### 现状

统一的 CSS 样式、公共 JS 函数（HTTP 请求、权限检查、格式化）、所有页面的侧边栏导航已实现。

### 接下来的目标

#### P0 — 全局 Toast 通知系统

当前提示用的是 `showError()` 写在指定的 DOM 容器里，有时提示出现在不可见区域：

```javascript
// 现状
showError('vehicle-alert', '错误信息');  // 要指定容器 ID

// 目标
toast.error('错误信息');   // 全局浮动提示
toast.success('操作成功');
toast.warning('注意...');
```

要求：
- 固定在页面右上角
- 3 秒自动消失
- 多条可堆叠
- 不同级别颜色不同

#### P0 — 表单验证统一

当前每个页面的表单验证逻辑分散在各处：

- [ ] 在 `common.js` 中提供统一验证工具
- [ ] `validate.required(value, '字段名')` — 非空
- [ ] `validate.phone(value)` — 手机号
- [ ] `validate.plate(value)` — 车牌号
- [ ] 统一显示红色边框 + 错误提示文字

#### P1 — 全局 Loading 状态

当前某些操作没有加载状态，用户点了按钮不知道是否在请求：

- [ ] 在 `common.js` 中提供 `showLoading()` / `hideLoading()`
- [ ] 页面级的全屏遮罩加载
- [ ] 按钮级的加载状态（按钮内显示 spinner + 禁用点击）
- [ ] 表格数据的骨架屏加载

#### P1 — 响应式布局完善

当前在手机上布局会乱：

- [ ] 侧边栏在小屏时自动折叠为汉堡菜单
- [ ] 表格在小屏时支持水平滚动
- [ ] 卡片在小屏时自适应宽度
- [ ] 弹窗在小屏时全屏显示

#### P1 — 页面间状态同步

当前不同页面间数据不同步（比如在 Dashboard 充值后去个人中心余额没更新）：

- [ ] 提供一个简单的状态管理（基于事件或 localStorage 变化监听）
- [ ] 页面切换时自动刷新关键数据（余额、车位状态）
- [ ] 或者用 BroadcastChannel API 实现跨标签页同步

#### P2 — 深色模式

- [ ] CSS 变量支持 dark theme
- [ ] 右上角切换按钮
- [ ] 记住用户偏好（localStorage）

#### P2 — 组件库化

将重复的 UI 模式抽象为可复用函数：

```javascript
// 表格组件
createTable('#container', {
    columns: ['车牌号', '入库时间', '费用'],
    data: records,
    onRowClick: (row) => { ... }
});

// 弹窗组件
modal.confirm('确定删除？', {
    onConfirm: () => deleteRecord(id)
});
```

## 各模块优先级总览

| 模块 | P0（近期） | P1（中期） | P2（远期） |
|------|-----------|-----------|-----------|
| **A 基础架构** | CI/CD 流水线、日志系统 | 配置增强、Docker | 单元测试、代码规范工具 |
| **B 用户权限** | 密码重置、Token 刷新 | 登录安全、注册增强 | 多因素认证、用户分组 |
| **C 停车核心** | 时长显示、在场车辆列表 | 多停车场、分页查询 | 车位地图、特殊车辆通道 |
| **D 计费余额** | 充值订单、阶梯可视化 | 计费预览、优惠券 | 月度账单、发票功能 |
| **E 增值功能** | 预约倒计时、报表导出 | 拦截日志、预约历史 | 收入预测、公告增强 |
| **F 前端基础** | Toast 通知、表单验证 | 全局 Loading、响应式、状态同步 | 深色模式、组件库 |
